哈希表

当我们遇到了要快速判断一个元素是否出现集合里（是否出现过）的时候，就要考虑哈希法。

其实就是空间换时间

常见的三种哈希结构

  • 数组
  • 集合（set）
  • 映射（字典，map）

数组

数组可以理解为一个简单的哈希表，而且是map，它的key是下标数字，value是数组元素

集合

在C++中，set提供以下三种数据结构，其底层实现以及优劣如下表所示：

集合 底层实现 是否有序 值是否可以重复 能否更改值 查询效率 增删效率
std::set 红黑树 有序 否 否 O(log n) O(log n)
std::multiset 红黑树 有序 是 否 O(logn) O(logn)
std::unordered_set 哈希表 无序 否 否 O(1) O(1)

std::unordered_set底层实现为哈希表，std::set 和std::multiset 的底层实现是红黑树，红黑树是一种平衡二叉搜索树，所以key值是有序的，且key不可以修改，改动key值会导致整棵树的错乱，所以只能删除和增加。

当我们要使用集合来解决哈希问题的时候，优先使用unordered_set，因为它的查询和增删效率是最优的；如果需要集合是有序的，那么就用set，如果要求不仅有序还要有重复数据的话，那么就用multiset。

映射

在C++中，map提供以下三种数据结构，其底层实现以及优劣如下表所示：

映射 底层实现 是否有序 值是否可以重复 能否更改值 查询效率 增删效率
std::map 红黑树 key有序 key不可重复 key不可修改 O(logn) O(logn)
std::multimap 红黑树 key有序 key可重复 key不可修改 O(log n) O(log n)
std::unordered_map 哈希表 key无序 key不可重复 key不可修改 O(1) O(1)

std::unordered_map 底层实现为哈希表，std::map 和std::multimap 的底层实现是红黑树。同理，std::map 和std::multimap 的key也是有序的（这个问题也经常作为面试题，考察对语言容器底层的理解）。

map 是键值对的数据结构，map中，对key有限制，对value没有限制，因为key的存储方式使用红黑树实现的。

其他语言例如：java里的HashMap ，TreeMap 都是一样的原理。可以灵活贯通。

虽然std::set和std::multiset 的底层实现基于红黑树而非哈希表，它们通过红黑树来索引和存储数据。不过给我们的使用方式，还是哈希法的使用方式，即依靠键（key）来访问值（value）。所以使用这些数据结构来解决映射问题的方法，我们依然称之为哈希法。std::map也是一样的道理。

这里在说一下，一些C++的经典书籍上 例如STL源码剖析，说到了hash_set hash_map，这个与unordered_set，unordered_map又有什么关系呢？实际上功能都是一样一样的， 但是unordered_set在C++11的时候被引入标准库了，而hash_set并没有，所以建议还是使用unordered_set比较好，这就好比一个是官方认证的，hash_set，hash_map 是C++11标准之前民间高手自发造的轮子。

set和map的常用方法

以下是set和map常用方法说明：

  1. find(key)

    • 功能：返回指向键 key 的迭代器；若不存在，返回 end()。

  2. count(key)

    • 功能：返回键 key 的出现次数（set 和 map 中结果为 0 或 1，multiset和multimap中结果是key的出现次数）。

  3. insert()

    • 功能：

      • set：插入单个值（insert(value)），返回 pair<iterator, bool>（成功时 bool=true）。
      • map：插入键值对（insert({key, value}) 或 insert(make_pair(key, value))），返回值同 set。

  4. operator[]（仅 map）

    • 功能：通过键直接赋值。若键不存在，自动插入新键值对（值默认初始化）。

  5. erase()

    • 功能：支持三种删除方式：

      • 按值/键删除：erase(key)，返回删除的元素数量（0 或 1）。
      • 按迭代器删除：erase(iterator)，返回指向下一元素的迭代器。
      • 按范围删除：erase(first_it, last_it)，删除区间 [first, last)。

242. 有效的字母异位词 - 力扣（LeetCode）

字母异位词是通过重新排列不同单词或短语的字母而形成的单词或短语，并使用所有原字母一次。

判定是否是字母异位词，需要判断两个方面

  • 出现的字母是否相同
  • 每个字母出现的次数是否相等

所以使用map<char, int>，key是字母，value是出现的次数

时间复杂度O(n)\Omicron(n)O(n)，空间复杂度O(n)\Omicron(n)O(n)

可以使用数组来简化复杂度，下标可以对应为字母，数组元素对应为出现的次数。只有26个字母，直接开一个长度为26的数组就好

时间复杂度O(n)\Omicron(n)O(n)，空间复杂度O(1)\Omicron(1)O(1)

349. 两个数组的交集 - 力扣（LeetCode）

本题就是检索在两个数组中都存在的元素，也就是元素存在性的判断，想到哈希表，这里只用来讨论存在性，所以使用集合

数组

首先考虑数组充当集合，题中数组元素大小为1-1000，完全可以接受，哈希数组使用下标为key，对应数组元素，哈希数组值为bool，标记数组元素是否出现

时间复杂度为O(n)\Omicron(n)O(n)，空间复杂度为O(1)\Omicron(1)O(1)

集合

再次考虑直接使用集合set

202. 快乐数 - 力扣（LeetCode）

见官方题解

总体思路：在不断循环计算sum的过程中，会陷入两种情况

  • sum最终收敛到1，不断循环
  • sum最终收敛到某些数字组成的环，不断循环

这时，只要循环计算中是否出现了重复的sum，就可以避免陷入无限循环（不管它是1循环还是其他数字组成的循环）

哈希表（集合）

容易想到集合，来判定当前的sum在之前的计算中是否出现过

时间复杂度为O(logn)\Omicron(logn)O(logn)，空间复杂度为O(logn)\Omicron(logn)O(logn)，证明和理解见官方题解

快慢指针

最终sum的计算进入了一个环（不管是1的循环还是其他数字组成的循环，它都是一个环），那么可以联想到快慢指针

这是一个重要的思路，判断循环想到快慢指针

时间复杂度为O(logn)\Omicron(logn)O(logn)，时间复杂度为O(1)\Omicron(1)O(1)，

时间复杂度证明和理解见官方题解，空间复杂度很好理解，因为这种方法没有额外的空间开销

1. 两数之和 - 力扣（LeetCode）

暴力解法

容易直接想到，两层循环寻找目标元素

哈希表，映射

问题可以这样想，在遍历到某个元素n的时候，可以查询target-n在之前遍历的元素中是否出现过，一旦出现过，就说明当前元素n和查询到的这个元素满足条件，返回即可。这种查询是否出现过的问题，容易想到哈希表，分析一下使用集合还是映射或者是数组：本题中返回的目标元素的下标数组，所以查询target-n这个元素的时候，不仅要判断存在性，还要找到它对应的下标，集合只能判断存在性，所以使用映射，键值对中键是元素，值是下标。可以使用数组吗？题目中元素给出−109<=nums[i]<=109-10^9 <= nums[i] <= 10^9−109<=nums[i]<=109，范围太大，不可以使用数组。

454. 四数相加 II - 力扣（LeetCode）

思路上，就是两数之和的升级版，依然使用哈希表（映射）

先对前两个数组的和进行哈希存储，得出它们可能的和与出现的次数

再对后两个数组的和进行哈希查询，找到目标，加上满足条件的元组次数

时间复杂度为O(n2)\Omicron(n^2)O(n2)，空间复杂度为O(n2)\Omicron(n^2)O(n2)，最快情况下，哈希表中存了n2n^2n2个不同的和

383. 赎金信 - 力扣（LeetCode）

本题采用哈希表（映射），因为key是字母，只有26个，所以采用数组，可能会有更小的开销

用数组作为哈希表（映射），下标是key，对应到字母，元素是value，对应到字符出现的次数

遍历magazine，得到hash数组，就是得到了每个字母在magazine中出现的次数

再遍历ransomNote，每次对当前字母的value减1，完成遍历后，如果hash数组中出现了负数，说明magazine中的字母不够组成ransonNote，返回false；否则，则最后返回true

时间复杂度为O(n)\Omicron(n)O(n)，空间复杂度为O(1)\Omicron(1)O(1)，hash数组空间开销为常量

15. 三数之和 - 力扣（LeetCode）

本题有两种解法，分别是哈希表和双指针，这里介绍哈希表的方法，在双指针的章节会给出双指针的解法

哈希表

下面的解法来自题解：哈希 + 枚举（简单思路）

  1. 参考 LeetCode 第1题「两数之和」的思路，先使用哈希表记录每个数字出现的频率。
  2. 使用双重循环遍历哈希表中任意两个不同的数（包括自身），计算它们的和 sum = a + b，再查找是否存在 c = -sum。
  3. 根据三数是否相同，分类判断是否有足够的频数来组成合法的三元组。
  4. 为了避免重复三元组，使用 set<vector> 对结果进行去重。

时间复杂度为O(n2)\Omicron(n^2)O(n2)，空间复杂度为O(n+m)\Omicron(n+m)O(n+m)，n是freq，m是result_set

这个解法简单有逻辑，且实际运行的时间和空间开销都胜过了代码随想录的写法，就学习这个就好

总结

遇见需要判断某个元素之前是走出现过的场景，就要想到哈希表。

哈希表又分为数组，集合，映射。首先应该根据存在性之外是否还有其他值需要记录选择集合还是映射，确定好之后根据数据规模判断数组的可用性，能用数组可以减少开销，不能使用则选用正常的set或者map
