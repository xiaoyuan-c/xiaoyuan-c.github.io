排序+双指针

首先，对数组进行排序，提供单调性，方便后续处理

记第一个数字为a，第二个数字为b，第三个数字为c

开始遍历，用一个for循环追踪a，然后a之后的数组元素就是双指针的搜索空间，也就是在[i+1, nums.size()-1]中利用双指针找出b和c

设置左右指针left和right，寻找b和c

  • 如果 a+b+c > 0，那就要和更小，根据单调性，让right左移
  • 如果 a+b+c < 0，那就要和更大，根据单调性，让left右移
  • 如果 a+b+c == 0，满足条件，三元组加入结果集

去重逻辑

题目中还要求找到的三元组不能重复，这就需要去重逻辑

针对a的去重

if (i > 0 && nums[i] == nums[i - 1]) continue;

如果当前的a，与上一轮的a相等，就跳过此次循环

  • 如果这时的a能找到满足条件的三元组[a,b,c]，那这个解在上一次循环i-1中已经得出，所以跳过此次循环
  • 为什么不与i+1比较？如果这样比较，会遗漏[a,a,c]这种解

针对b和c的去重

已经找到满足条件的a,b,c后，要避免重复的b和c，所以让left和right都向数组中间移动，直到元素不再重复

  • 时间复杂度O(n2)O(n^2)O(n2)，排序O(logn)O(logn)O(logn)，因为双指针O(n)O(n)O(n)，所以查找是O(n2)O(n^2)O(n2)，排序加上查找就是O(n2)O(n^2)O(n2)
  • 空间复杂度O(1)O(1)O(1)

18. 四数之和 - 力扣（LeetCode）

逻辑与三数之和几乎完全相同

但是三数之和中target是固定为0的，而这里不是固定的，所以剪枝的时候要判定是非负整数才能剪枝，因为，如果target是负数，即使目前的a大于target，但是后续可能存在负数bcd，让和变小，可能与target相等，这种情况就不能被剪枝去掉。

N数之和

目前有两种解法，一种是哈希，一种是双指针

哈希见哈希的章节笔记，利用map构建频次表去重和找到结果。这里总结以下双指针。

1. 两数之和 - 力扣（LeetCode）一般使用哈希解决，但是这题要求返回的是下标的二元组，如果要求返回的是元素，那一样可以使用双指针解决，排序数组，设置left从左开始，right从右开始，如果和大于target，right左移；如果和小于target，left右移。

15. 三数之和 - 力扣（LeetCode）和18. 四数之和 - 力扣（LeetCode）的双指针思路几乎相同

  • 15. 三数之和 - 力扣（LeetCode）中，用for循环确定a，left和right双指针在[i + 1, nums.size()-1]中搜索b和c

  • 18. 四数之和 - 力扣（LeetCode）中，用两层for循环确定a和b，left和right双指针在[j+1, nums.size()-1]中搜索c和d

  • 本质上，都是用双指针的解法减少了一层for循环，让时间复杂度降低一个次方，对于三数之和使用双指针法就是将原本暴力O(n3)O(n^3)O(n3)的解法，降为O(n2)O(n^2)O(n2)的解法，四数之和的双指针解法就是将原本暴力O(n4)O(n^4)O(n4)的解法，降为O(n3)O(n^3)O(n3)的解法。

  • 同样的道理，可以扩展到五数之和，n数之和等

总结

其实很多情况下，双指针法都是用一个循环解决了两个循环的事情，一般就是将O(n2)O(n^2)O(n2)的时间复杂度，降为 O(n)O(n)O(n) 。
